## 🎨 v1.6.0 新規コンポーネント提案（優先順位順）

### 🥇 最優先（MUST HAVE）- 5コンポーネント

#### 1. **Sidebar Navigation** (Tier 1)
- **推定行数**: 500-600行
- **用途**: ドキュメントサイト、管理画面、ダッシュボード
- **機能**:
  - 固定/折りたたみ可能サイドバー
  - マルチレベル階層メニュー（最大4階層）
  - 現在位置のハイライト
  - 折りたたみアニメーション
  - モバイル対応ドロワー
  - アイコン + テキスト/アイコンのみモード
  - ミニサイドバーモード（ホバーで展開）
- **既存との差別化**: Navigationはヘッダー特化、Sidebarは縦長メニュー専用
- **必要性**: ドキュメントサイト100%必須、管理画面90%必須

#### 2. **Kanban Board** (Tier 1)
- **推定行数**: 600-700行
- **用途**: プロジェクト管理、タスク管理、ワークフロー
- **機能**:
  - 列（カラム）レイアウト（2-6列）
  - カードコンポーネント（タイトル、説明、ラベル、アバター）
  - ドラッグ&ドロップ視覚フィードバック（CSS）
  - カラムヘッダー（タイトル、カウント、アクション）
  - 空のカラム状態
  - スクロール可能な長いカラム
  - カードのホバー/フォーカス状態
  - レスポンシブ（モバイルで縦積み）
- **既存との差別化**: Steps/Timelineとは異なるレイアウト
- **必要性**: プロジェクト管理サイト必須、SaaS管理画面80%必須

#### 3. **Code Block** (Tier 1)
- **推定行数**: 400-500行
- **用途**: ドキュメントサイト、技術ブログ、APIリファレンス
- **機能**:
  - シンタックスハイライト準備（クラスベース）
  - 行番号表示
  - ファイル名/言語表示ヘッダー
  - コピーボタン領域
  - インラインコード vs ブロックコード
  - スクロール可能な長いコード
  - テーマ（Light/Dark）対応
  - 行ハイライト（特定行の強調）
- **既存との差別化**: Typographyの`<code>`では機能不足
- **必要性**: ドキュメントサイト100%必須、技術ブログ95%必須

#### 4. **Dashboard Widget/Card** (Tier 1)
- **推定行数**: 450-550行
- **用途**: 管理画面、ダッシュボード、統計表示
- **機能**:
  - 統計カード（数値、タイトル、アイコン、トレンド矢印）
  - チャート埋め込み領域
  - アクションメニュー（ヘッダー右上）
  - フッターリンク
  - 複数サイズ（小、中、大）
  - カラーバリアント（primary, success, warning, error）
  - ローディング状態
  - 空データ状態
- **既存との差別化**: Cardは汎用、Widgetは統計特化
- **必要性**: ダッシュボード100%必須、管理画面90%必須

#### 5. **Filter/Search Panel** (Tier 1)
- **推定行数**: 500-600行
- **用途**: 検索機能、フィルター機能、商品絞り込み
- **機能**:
  - 検索入力フィールド
  - フィルターグループ（カテゴリ別）
  - チェックボックス/ラジオボタングループ
  - 価格レンジスライダー（視覚的準備）
  - 日付範囲フィルター
  - タグ/バッジでの選択済み表示
  - クリアボタン
  - 折りたたみ可能セクション
  - モバイル対応モーダル/ドロワー
- **既存との差別化**: Dropdown単体では複雑なフィルター不可
- **必要性**: ECサイト100%、検索サイト95%、管理画面80%必須

## 🏗️ モジュール型構造強化提案

### 現在の課題
1. **コンポーネント間の依存関係が不明確**
   - 例: Cardはどのコンポーネントと組み合わせ可能か？
2. **バリアント（変種）の命名規則が統一されていない**
   - 例: `--primary`, `--bordered`, `--lg` の混在
3. **共通パターンの抽出が不十分**
   - 例: すべてのコンポーネントに`--sm`, `--lg`が必要か？

### 強化策

#### 1. **コンポーネント分類システム**
```
Adlaire Architect Components
├── 🎯 Core (基盤)
│   ├── Typography
│   ├── Buttons
│   ├── Forms
│   └── Grid
├── 📦 Layout (レイアウト)
│   ├── Sidebar Navigation ← NEW
│   ├── Navigation
│   ├── Breadcrumb
│   └── Divider
├── 📊 Data Display (データ表示)
│   ├── Table
│   ├── List Group ← NEW
│   ├── Cards
│   ├── Dashboard Widget ← NEW
│   ├── Stats Counter ← NEW
│   └── Badge
├── 🗂️ Content (コンテンツ)
│   ├── Accordion
│   ├── Tabs
│   ├── Carousel
│   ├── Lightbox ← NEW
│   ├── Timeline
│   ├── Code Block ← NEW
│   └── Callout ← NEW
├── 📝 Forms Extended (フォーム拡張)
│   ├── Date Picker
│   ├── Dropdown
│   ├── Filter Panel ← NEW
│   └── Search Box (Filter Panelに統合)
├── 🔔 Feedback (フィードバック)
│   ├── Alert
│   ├── Toast
│   ├── Modal
│   ├── Progress
│   ├── Skeleton Loader ← NEW
│   ├── Empty State ← NEW
│   └── Tooltip
├── 🚀 Interactive (インタラクティブ)
│   ├── Kanban Board ← NEW
│   ├── Steps
│   ├── Pagination
│   └── Calendar
├── 🌐 Global (グローバル)
│   ├── Mega Menu ← NEW
│   ├── Language Switcher ← NEW
│   └── Avatar
└── 💼 Business (ビジネス特化)
    ├── Pricing Table ← NEW
    └── Testimonial ← NEW
```

#### 2. **共通サイズシステム標準化**
すべてのコンポーネントに以下のサイズバリアントを適用：
```css
.aa-{component}--xs   /* Extra Small (必要な場合のみ) */
.aa-{component}--sm   /* Small */
.aa-{component}        /* Base (デフォルト、modifierなし) */
.aa-{component}--lg   /* Large */
.aa-{component}--xl   /* Extra Large (必要な場合のみ) */
```

#### 3. **共通カラーシステム標準化**
すべてのコンポーネントに以下のカラーバリアントを適用（該当する場合）：
```css
.aa-{component}--primary
.aa-{component}--secondary
.aa-{component}--success
.aa-{component}--warning
.aa-{component}--error
.aa-{component}--info
.aa-{component}--neutral
```

#### 4. **共通状態システム標準化**
```css
.aa-{component}--active
.aa-{component}--disabled
.aa-{component}--loading
.aa-{component}--hover (hover疑似クラスと併用)
.aa-{component}--focus (focus疑似クラスと併用)
```

#### 5. **コンポーネントヘッダー標準化**
すべての新規コンポーネントに以下のヘッダーを付与：
```css
/**
 * {Component Name} Component
 * Version: 1.6.0
 * Category: {Core|Layout|Data Display|Content|Forms Extended|Feedback|Interactive|Global|Business}
 * 
 * A {brief description}
 *
 * Features:
 * - Feature 1
 * - Feature 2
 * - ...
 *
 * Dependencies: {None|Component A, Component B}
 * Composable with: {Component X, Component Y}
 * 
 * Sizes: xs, sm, base, lg, xl
 * Colors: primary, secondary, success, warning, error, info
 * States: active, disabled, loading
 * 
 * Dark mode: ✅ Full support
 * Responsive: ✅ Mobile-first
 * Accessibility: ✅ WCAG 2.1 AAA
 * Keyboard navigation: ✅ Arrow keys, Tab, Enter, Escape
 */
```

#### 6. **デザイントークン拡張**
新規トークンを`src/tokens/root.css`に追加：
```css
/* Component-specific tokens */
--adlaire-sidebar-width: 280px;

--adlaire-code-block-bg: #1e1e1e;
--adlaire-widget-padding: var(--adlaire-space-4);
/* ... */
```

8. **Print Styles** - 印刷時の最適化
9. **Reduced Motion** - アニメーション無効化対応

```

各ファイルの構成:
1. ヘッダーコメント（Component情報）

4. Size Variants
5. Color Variants
6. State Variants
7. Dark Mode Support

11. Print Styles
```
